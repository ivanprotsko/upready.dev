**Определение:**
- Default: written communication
- Sync interruptions = exceptions
- Focus time protection
- Document ideas vs just discussing
- Global collaboration без inconvenient hours

- **Focus time:** Developers need 2-4 hour blocks

- **Flexibility:** Work during most productive hours
- **Deep work facilitation**

**Как работает:**
- Rich artifacts: documents, diagrams, comments, videos
- Respondent controls when to consume & reply
- Discussion where work happens (Figma, GitHub PRs)
- Context не теряется в isolated chats

#### Практические стратегии Async-First

**"Quiet Hours" Setup:** Specific deep work blocks, no interruptions, schedule during peak productivity

**Tools для Async:**
- **Loom:** Quick video updates, walkthroughs
- **Notion:** Documentation, written updates
- **Slack:** Async messaging, no @here/@channel abuse
- **GitHub PRs:** Code review comments
- **Figma:** Design feedback comments
- **Linear:** Issue updates and comments

**Default Communication Hierarchy:**
1. **Written updates** (Notion, Slack, Linear)
2. **Async video** (Loom walkthroughs)
3. **Voice messages** (if needed)
4. **Scheduled calls** (last resort)

**When to use Sync:**
- Kickoff calls (rare)
- Major decisions requiring rapid back-and-forth
- Emergency blockers
- Relationship building (occasional)

**Client Expectations Setting:**
- Clear communication: "async-first workflow"
- Response time SLAs (e.g., 24 hours)
- Emergency protocol (for urgent issues)
- Regular updates schedule

**Benefits:**
- Eliminates calendar fragmentation
- Maximizes focus time
- Better documentation (everything written/recorded)
- Timezone flexibility
- Client convenience
